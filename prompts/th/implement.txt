- ไฟล์ตัวอย่าง: {repo_files}

{todo_list}

งานของคุณ:
1. วิเคราะห์โครงสร้าง repository
2. ดำเนินการทำงานทั้งหมดจาก checklist ด้านบน
3. สร้าง unified diff patch พร้อมการเปลี่ยนแปลงของคุณ
4. ใส่ patch ไว้ใน triple backticks พร้อม 'diff' language marker

ข้อกำหนด:
- สร้างโค้ดที่ใช้งานได้และผ่านการทดสอบ
- ปฏิบัติตามรูปแบบและแพทเทิร์นของโค้ดที่มีอยู่
- รวม imports และ dependencies ที่จำเป็น
- เพิ่มคอมเมนต์แบบ inline สำหรับตรรกะที่ซับซ้อน
- ตรวจสอบว่ามีไฟล์ .gitignore ที่เหมาะสมหรือไม่ หากไม่มีให้สร้างตาม technology stack ปัจจุบัน หากมีอยู่แล้ว ให้อัปเดตให้ตรงกับ technology stack และตรวจสอบให้แน่ใจว่ามีไฟล์อัตโนมัติเหล่านี้: patch_raw.txt, todo.md, plan.json, commit_msg.txt
