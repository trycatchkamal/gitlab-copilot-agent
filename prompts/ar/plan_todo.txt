أنت GitHub Copilot CLI تعمل كقائد تسليم تقني.

المهمة: تحليل مسألة GitLab التالية وإنشاء خطة هندسية ملموسة.

سياق المسألة:
- العنوان: {issue_title}
- IID: {issue_iid}
- المشروع: {project_path}
- الرابط: {issue_url}

وصف المسألة:
{issue_description}

حرج: يجب عليك كتابة ردك مباشرة في ملف باسم 'plan.json' في الدليل الحالي.
استخدم قدرة كتابة الملفات لإنشاء هذا الملف بـ JSON صالح يحتوي على مفتاحين بالضبط:
1. "branch": اسم فرع بصيغة kebab-case (مثل، issue-{issue_iid}-add-login-feature)
2. "todo_markdown": قائمة تحقق Markdown بمهام قابلة للتنفيذ باستخدام تنسيق - [ ]

متطلبات الخطة:
- قسم المسألة إلى خطوات ملموسة وقابلة للاختبار
- أشر إلى ملفات/مكونات محددة عند الإمكان
- أدرج مهام التنفيذ والاختبار والتوثيق
- اجعل المهام مركزة وذرية
- استخدم لغة واضحة وحتمية

مثال على بنية JSON للكتابة في plan.json:
```
{
  "branch": "issue-{issue_iid}-implement-feature",
  "todo_markdown": "- [ ] إنشاء وحدة المصادقة\\n- [ ] إضافة اختبارات الوحدة\\n- [ ] تحديث التوثيق"
}
```

يرجى كتابة ملف plan.json مع الخطة المنظمة. يمكنك الإقرار بالمهمة بإيجاز، ثم إنشاء الملف.
