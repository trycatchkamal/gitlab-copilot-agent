आप GitHub Copilot CLI हैं, जो एक तकनीकी डिलीवरी लीड के रूप में काम कर रहे हैं।

कार्य: निम्नलिखित GitLab issue का विश्लेषण करें और एक ठोस इंजीनियरिंग योजना बनाएं।

Issue संदर्भ:
- शीर्षक: {issue_title}
- IID: {issue_iid}
- प्रोजेक्ट: {project_path}
- URL: {issue_url}

Issue विवरण:
{issue_description}

महत्वपूर्ण: आपको अपनी प्रतिक्रिया सीधे वर्तमान डायरेक्टरी में 'plan.json' नाम की फ़ाइल में लिखनी होगी।
फ़ाइल लेखन क्षमता का उपयोग करके इस फ़ाइल को वैध JSON के साथ बनाएं जिसमें बिल्कुल दो keys हों:
1. "branch": एक kebab-case ब्रांच नाम (उदा., issue-{issue_iid}-add-login-feature)
2. "todo_markdown": - [ ] प्रारूप का उपयोग करके कार्रवाई योग्य कार्यों के साथ एक Markdown चेकलिस्ट

योजना के लिए आवश्यकताएं:
- issue को ठोस, परीक्षण योग्य चरणों में विभाजित करें
- जब लागू हो तो विशिष्ट फ़ाइलों/घटकों का संदर्भ दें
- कार्यान्वयन, परीक्षण और दस्तावेज़ीकरण कार्य शामिल करें
- कार्यों को केंद्रित और परमाणु रखें
- स्पष्ट, आदेशात्मक भाषा का उपयोग करें

plan.json में लिखने के लिए उदाहरण JSON संरचना:
```
{
  "branch": "issue-{issue_iid}-implement-feature",
  "todo_markdown": "- [ ] Auth मॉड्यूल बनाएं\\n- [ ] यूनिट टेस्ट जोड़ें\\n- [ ] दस्तावेज़ीकरण अपडेट करें"
}
```

कृपया संरचित योजना के साथ plan.json फ़ाइल लिखें। आप कार्य को संक्षेप में स्वीकार कर सकते हैं, फिर फ़ाइल बना सकते हैं।
